Computer Engineering 288, Search Roomba

This was a team project as a part of the CPRE 288 class I took my sophomore year at ISU. For this project, we did a small but full project cycle. This included figuring out the problem or issue we were solving, possible solutions to the problem, and finally creating a proof-of-concept of a solution to this problem. The problem we aimed to solve was how to remotely explore possibly dangerous areas for things like search and rescue after a disaster. The solution we came up with was a remote-controlled bot that was of a smaller size so it could get through tight areas. With the Roomba that we were given to accomplish this, we coded the micro-controller running the Roomba using C to access the registers and sensor data. In the end, we drove the Roomba through a small series of obstacles until it found its goal without being able to see the course or the Roomba. For this run, I was the driver of the Roomba and was responsible for ensuring we were in the goal before we signaled that we thought we were at the end. A video of this can be viewed here:

C/C++ Roguelike Video Game, Computer Science 327

This is one of the favorite projects I have done. It consisted of week to week work throughout a semester implementing specified features of a roguelike game in C and eventually C++. We started from the basics of randomly generating dungeons and finished with a game that uses ncurses to display a dungeon that is filled with monsters and items which each have their own characteristics. This project helped me gain a lot of experience in refactoring and adding onto someone else's code because each week the professor would post his solution to the assignment and allowed us to build off his for the next assignment. Having his code available also exposed me to new ways of doing things in C and C++ that he learned through his work experience. I also got a lot of practice debugging issues using gdb and Valgrind because I used them to check for memory leaks and get more information on any faults my game had. If you would like to try the game out, take a look at my GitHub page under COMS327Spring2019 or at the direct link below. Each folder is a different version of the code with newer versions building off the previous ones, except for version 2.0 which is a hangman/wheel of fortune solver I did at the end of the semester. Look at the READMEs for more information on how to run the game, mostly you just need to have g++ and make to run the makefile. Then you can execute ./rlg327 to play the game.
